## 1.[[Medium]2.两数相加](https://leetcode.cn/problems/add-two-numbers/)		

​	关键在于考虑到多种特殊情况的处理，例如刚开始head指针指向NULL，最后carry有进位的保存。
​	最主要还是暴力解法sum超出数据大小可存储上限，所以不应该总体求出和最后再拆分每一位，而是要将总的求和拆分成一次次求和，每次都保存一位

```c
struct ListNode* addTwoNumbers(struct ListNode* l1, struct ListNode* l2) {
	struct ListNode* head = NULL, * tail = NULL;
	int sum = 0, carry = 0;
	while (l1 || l2) {
		int a = l1 != NULL ? l1->val : 0;
		int b = l2 != NULL ? l2->val : 0;
		sum = a + b + carry;
		//if (sum == 0) {	处理错误，和为0不需要进行特殊处理，应当做一般情况
		if (head == NULL) {
			head = tail = malloc(sizeof(struct ListNode));
			tail->val = sum % 10;
			tail->next = NULL;
		}
		else {
			tail->next = malloc(sizeof(struct ListNode));
            tail = tail->next;
			tail->val = sum % 10;
			tail->next = NULL;
		}
		carry = sum / 10;
		if (l1 != NULL) {
			l1 = l1->next;
		}
		if (l2 != NULL) {
			l2 = l2->next;
		}
	}
    if(carry > 0)
    {
        tail->next = malloc(sizeof(struct ListNode));
        tail->next->val = carry;
        tail->next->next = NULL;
    }
	return head;
}
```

## 2.[[Easy]88.合并两个有序数组](https://leetcode.cn/problems/merge-sorted-array/)

---------------*要求：不能引用额外数组空间*---------------

核心在于仅优先插入数组2当前小于数组1当前值的值，若大于暂时不管，此时遍历数组1。（因为仅知道大概位置即数组1当前值之后，若要确定具体位置(因为数组1当前位置紧邻之后的值可能小于当前数组2的值)则会增加工作量，并且，即使不管之后也能自动处理即暂且跳过数组1当前值即i++）==例如a1[1,2,3,4,5]，a2[6,7,8]，显然将6插入到1之后是不对的，但是a1,a2调换一下1插入6之前肯定对，因为是非严格递增数组，小于前面的值则一定小于后面所有的值==

**法一：从前往后合并(逻辑较混乱)——一般**

```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

/*
给你两个按非递减顺序排列的整数数组nums1和nums2，另有两个整数m和n，分别表示nums1和nums2中的元素数目。
注意：最终，合并后数组不应由函数返回，而是存储在数组nums1中。为了应对这种情况，nums1的初始长度为m+n，
其中前m个元素表示应合并的元素，后n个元素为0，应忽略。nums2的长度为n。
*/

void merge(int* nums1, int nums1Size, int m, int* nums2, int nums2Size, int n) {
    int i = 0;  // nums1 的当前位置
    int j = 0;  // nums2 的当前位置

    //！！！核心在于仅优先插入数组2当前小于数组1当前值的值，若大于暂时不管
    //！！！（因为仅知道大概位置即数组1当前值之后，若要确定具体位置则会增加工作量
    //！！！并且，即使不管之后也能自动处理即暂且跳过数组1当前值即i++）

    while (j < n) {
        // 如果 nums1 已经处理完所有原始有效元素，直接把 nums2 剩余元素放到后面
        if (i >= m + j) {
            nums1[i] = nums2[j];
            i++;
            j++;
            continue;
        }

        // 如果当前 nums1 的元素大于 nums2 的元素，需要插入
        if (nums1[i] > nums2[j]) {
            // 把从 i 到 m+j-1 的元素后移一位
            for (int k = nums1Size; k > i ; k--) {
                nums1[k] = nums1[k - 1];
            }
            nums1[i] = nums2[j];
            j++;  // nums2 指针前进
            i++;  // nums1 指针前进（因为插入了新元素）
        }
        else {
            // nums1[i] <= nums2[j]，直接跳过 nums1 的这个元素
            i++;
        }
    }
}
/*
1, 2 ,!2 ,3 ,0 ,0
2, !5, 6
1, 2, 2, 5, 3, 0
*/
int main(void) {
	int nums1[] = { 1, 2, 3, 0, 0 ,0 };
	int nums2[] = { 2, 5, 6 };

	int m = 3, n = 3;
	merge(nums1, 6, m, nums2, 3, n);
	for (int i = 0; i < 6; i++) {
		printf("%d\n", nums1[i]);
	}

	return 0;
}
```

## 3.[[Easy]9.回文数](https://leetcode.cn/problems/palindrome-number)		

**法一：先求数字位数，根据位数一步步拆解首位与末位并比对——一般**

```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <stdbool.h>
#include <math.h>

/*
给你一个整数 x ，如果 x 是一个回文整数，返回 true ；否则，返回 false 。

回文数是指正序（从左向右）和倒序（从右向左）读都是一样的整数。

例如，121 是回文，而 123 不是。


示例 1：

输入：x = 121
输出：true
示例 2：

输入：x = -121
输出：false
解释：从左向右读, 为 -121 。 从右向左读, 为 121- 。因此它不是一个回文数。
示例 3：

输入：x = 10
输出：false
解释：从右向左读, 为 01 。因此它不是一个回文数。


提示：

-231 <= x <= 231 - 1


进阶：你能不将整数转为字符串来解决这个问题吗？
*/

bool main(void) {
	int x;
	scanf("%d", &x);
	int digits = 0;
	//特殊处理,0和负数
	if (x == 0)
		return true;
	else if (x < 0)
		return false;
	else {
		//首先计算这个十进制数的位数
		int t = x;
		while (t) {
			t /= 10;
			digits++;
		}
		while (x) {
			int high, low;
			high = x / (pow(10, digits - 1));
			low = x % 10;
			x -= high * pow(10, digits - 1);
			x /= 10;
			digits -= 2;
			if (high != low)
				return false;
		}
		return true;
	}
}
```

```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <stdbool.h>
/*
前置知识一：如何遍历一个整数
如果不把 x 转成字符串，要怎么做？

我们可以不断地取 x 的最低位（模 10），去掉 x 的最低位（除以 10），直到 x=0。

例如 x=123：

通过 xmod10 取到个位数 3，然后把 x 除以 10（下取整），得到 x=12。
再次 xmod10 取到十位数 2，然后把 x 除以 10（下取整），得到 x=1。
最后 xmod10 取到百位数 1，然后把 x 除以 10（下取整），得到 x=0。此时完成遍历，退出循环。
前置知识二：如何反转一个整数
比如我们现在有一个数 56，如何把 7 加到 56 的末尾？

把 56 乘以 10，再加上 7，就得到了 567。

一般地，如果要把数字 b 加到整数 a 的末尾，我们可以计算 a⋅10+b。

如果要把 x=123 反转，我们可以按照前置知识一中的方法，从低到高遍历 x 的每一位，即 3,2,1。

初始化 rev=0，依次把 3,2,1 加到 rev 的末尾，即：

更新 rev 为 rev⋅10+3=3。
更新 rev 为 rev⋅10+2=32。
更新 rev 为 rev⋅10+1=321。
最终得到了 x 反转后的结果 rev=321。
本题思路
暴力想法是，根据前置知识二，把 x 整体反转，得到 rev，然后判断 rev 和 x 是否相等。

但实际上，对于回文数来说，只需判断 x 的左半部分是否等于右半部分就行。

*/

//反转一半整数
bool isPalindrome(int x) {
    if (x < 0 || x > 0 && x % 10 == 0) {
        return false;
    }
    int rev = 0;
    while (rev < x / 10) {
        rev = rev * 10 + x % 10;
        x /= 10;
    }
    return rev == x || rev == x / 10;
}

    printf("Please enter a number:");
    scanf("%d", &x);
```

# ==三道经典滑动窗口入门级模板题==

滑动窗口总结：（目前仅考虑窗口左右端口均右移的问题）
外层**for**循环对应**窗口过小即让窗口右端右移**，内层**while**循环处理**窗口过大即让窗口左端右移**
for中参数**(int right = 0;  right < ?; right ++)**        while中参数**(临界条件：保证跳出时刚好满足条件，即正好窗口左端无法右移)**

```
滑动窗口（可变长度）能用来解决子数组和问题，依赖一个关键性质：
							当窗口扩张时，和增加；当窗口收缩时，和减少。
这意味着：
如果数组中只有正数，那么随着窗口右边界右移，子数组和单调递增
因此当当前窗口和超过目标值 k 时，我们可以确定地收缩左边界来减少和
这保证了滑动窗口的双指针移动方向是单向的，时间复杂度 O(n)
```

## 4.[[Medium]3.无重复字符的最长子串](https://leetcode.cn/problems/longest-substring-without-repeating-characters)——“滑动窗口”


```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>
#include <stdlib.h>
#include <time.h>

/*
给定一个字符串s，请你找出其中不含有重复字符的最长子串的长度。
*/

int lengthOfLongestSubstring(char* s) {
    int ans = 0, left = 0;
    int cnt[128] = { 0 }; // 也可以用哈希表，这里为了效率用的数组
    for (int right = 0; s[right]; right++) {
        // char c = s[right];
        // cnt[c]++;
        cnt[s[right]] ++;
        // 窗口内有重复字母—cnt[c]<=1退出，即某字符在一个数量基础上每多一个就在最左边去掉一个
        //！！并且若是移去的不是窗口右端的字母则会一直移出直到保证当前窗口内的所有种字母都只有1个
        //！！此举有效解决了相同字符紧邻的情况如"xwwbw"：如下折叠注释
        /*right = 2，字符 s[2] = 'w'
            cnt['w']++→ cnt['w'] = 2 ✅ 出现重复

            进入 while 循环（因为 cnt['w'] > 1）

            移除 s[left] = s[0] = 'x'
            cnt['x']--→ cnt['x'] = 0
            left++ → left = 1
            cnt['w'] 还是 2（没变，因为移除的是 'x'）
            继续 while：cnt['w'] = 2 > 1
            移除 s[left] = s[1] = 'w'
            cnt['w']--→ cnt['w'] = 1
            left++ → left = 2
            现在 cnt['w'] = 1，退出 while
            窗口长度 = right - left + 1 = 2 - 2 + 1 = 1

            ans = max(2, 1) = 2 （不变）

            窗口内容：[w]（s[2] 一个）*/
        while (cnt[s[right]] > 1) { 
            cnt[s[left]]--; // 移除窗口左端点字母
            left++; // 缩小窗口
        }
        ans = ans > (right - left + 1) ? ans : (right - left + 1); // 更新窗口长度最大值
    }
    return ans;
}

int main(void) {
	char s[128];	//字符串
	int i = 0;
	while ((s[i] = getchar()) != '\n')
       i++; 
	s[i] = '\0';

	printf("%d\n", lengthOfLongestSubstring(s));

	return 0;
}
```

## 5.[[Medium]209.长度最小的子数组 ](https://leetcode.cn/problems/minimum-size-subarray-sum/description/)——“滑动窗口”

```c
/*
给定一个含有 n 个正整数的数组和一个正整数 target 。找出该数组中满足其总和大于等于 target 的长度最小的 子数组 [numsl, numsl+1, ..., numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0 。
*/

int minSubArrayLen(int target, int* nums, int numsSize) {
    int sum = 0, left = 0;
    int ans = numsSize + 1; //！！找最小，则初值设置为最大
    for(int right = 0 ; right < numsSize; right ++){    //外层循环处理sum小于target
        sum += nums[right];
        while(sum - nums[left] >= target){   //内层循环处理若是去掉窗口左端值后sum大于target
            sum -= nums[left];
            left ++;
        }
        if(sum >= target)
            ans = (right - left + 1) < ans ? (right - left + 1) : ans;
    }
    if(ans < numsSize + 1)
        return ans;
    else return 0;
}
```

## 7.[[Medium]153. 寻找旋转排序数组中的最小值](https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/description/)——“开区间二分”

1. 由数组下标(索引)的特殊可知，每次取mid=(low+high)/2;时，mid所指的位置一定在正中位置或者正中偏左1位置，即mid左半段一定少于右半段。
2. 二分的范围可以是开区间 (−1,*n*−1)，也就是闭区间 [0,*n*−2]。

```c
/*
已知一个长度为n的数组，预先按照升序排列，经由1到n次旋转后，得到输入数组。例如，原数组nums=[0,1,2,4,5,6,7]在变化后可能得到：
•	若旋转4次，则可以得到[4,5,6,7,0,1,2]
•	若旋转7次，则可以得到[0,1,2,4,5,6,7]
注意，数组[a[0],a[1],a[2],...,a[n-1]]旋转一次的结果为数组[a[n-1],a[0],a[1],a[2],...,a[n-2]]。
给你一个元素值互不相同的数组nums，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的最小元素。
你必须设计一个时间复杂度为O(logn)的算法解决此问题。
n == nums.length
1 <= n <= 5000
-5000 <= nums[i] <= 5000
nums 中的所有整数！互不相同！
nums 原来是一个升序排序的数组，并进行了 1 至 n 次旋转
*/

/*
设 x=nums[mid] 是现在二分取到的数。
我们需要判断 x 和数组最小值的位置关系，谁在左边，谁在右边？
把 x 与最后一个数 nums[n−1] 比大小：
如果 x>nums[n−1]，那么可以推出以下结论：
    nums 一定被分成左右两个递增段；
    第一段的所有元素均大于第二段的所有元素；
    x 在第一段。
    最小值在第二段。
    所以 x 一定在最小值的左边。
如果 x≤nums[n−1]，那么 x 一定在第二段。（或者 nums 就是递增数组，此时只有一段。）
    x 要么是最小值，要么在最小值右边。
    所以，只需要比较 x 和 nums[n−1] 的大小关系，就间接地知道了 x 和数组最小值的位置关系，从而不断地缩小数组最小值所在位置的范围，二分找到数组最小值。

细节
下面代码用的开区间二分，用其他二分写法也是可以的。
二分的范围可以是 (−1,n−1)，也就是闭区间 [0,n−2]。
这是因为，如果 nums[n−1] 是数组最小值，那么 nums 分成两段，第一段 [0,n−2]，第二段 [n−1,n−1]，且第一段的所有数都大于 nums[n−1]。每次 x 和 nums[n−1] 比大小，一定是 x>nums[n−1]。这意味着每次二分更新的都是 left，那么循环结束后，答案自然就是 n−1 了。

注：这里有两个概念「二分范围」和「答案范围」。答案确实可以等于 n−1，但对于二分来说，代码中的 if (nums[mid] < nums[n - 1]) 在 mid=n−1 的时候一定不成立，我们可以直接知道 n−1 是蓝色（根据视频中的红蓝染色法），所以 n−1 无需在二分区间中。
*/

int findMin(int* nums, int numsSize) {
    int left = -1, right = numsSize - 1; // 开区间 (-1, n-1)
    while (left + 1 < right) { // 开区间不为空
        int mid = left + (right - left) / 2; // 不易越界
        if (nums[mid] < nums[numsSize - 1])
            right = mid;
        else 
            left = mid;
    }
    return nums[right];
}
```

```c

    int len_cur = numsSize, i = 0;
    while(i < len_cur){
        if(nums[i] == val){
            for(int j = i; j < len_cur - 1; j++)
                nums[j] = nums[j+1];
```

```c
    for (int right = 0; right < numsSize; right++) { // 在原数组中的遍历指针
```

**法三：双指针优化（初始位于左右两端）——最优**

```c
/*
思路
	如果要移除的元素恰好在数组的开头，例如序列 [1,2,3,4,5]，当 val 为 1 时，我们需要把每一个元素都左移一位。注意到题目中说：「元素的顺序可以改变」。实际上我们可以直接将最后一个元素 5 移动到序列开头，取代元素 1，得到序列 [5,2,3,4]，同样满足题目要求。这个优化在序列中 val 元素的数量较少时非常有效。
	实现方面，我们依然使用双指针，两个指针初始时分别位于数组的首尾，向中间移动遍历该序列。

算法：
	如果左指针 left 指向的元素等于 val，此时将右指针 right 指向的元素复制到左指针 left 的位置，然后右指针 right 左移一位。如果复制过来的元素恰好也等于 val，可以继续把右指针 right 指向的元素的值复制过来（左指针 left 指向的等于 val 的元素的位置继续被覆盖），直到左指针指向的元素的值不等于 val 为止。
	当左指针 left 和右指针 right 重合的时候，左右指针遍历完数组中所有的元素。
	这样的方法两个指针在最坏的情况下合起来只遍历了数组一次。与方法一不同的是，方法二避免了需要保留的元素的重复复制操作。
*/

int removeElement(int* nums, int numsSize, int val) {
    int left = 0, right = numsSize; // 初始两指针分别位于左右两端
    while (left < right) {
        if (nums[left] == val) {
            nums[left] = nums[right - 1];
            right--; // 左指针不动，因为得继续判断空降过来的元素
        } else {
            left++;
        }
    }
    return left; // 最终位于新数组下标(索引)右端+1的位置，即为新数组元素个数
}
```

```c
int strStr(char* haystack, char* needle) {
    int haystack_size = 0, needle_size = 0;
    for(int i = 0; haystack[i]; i++)
        haystack_size++;
    for(int i = 0; needle[i]; i++)
        needle_size++;

        int ans = i;
        for (int j = 0; j < needle_size; j++) {

// ---------------------------KMP算法-----------------------------
// next数组
int *build_next(char* needle, int len) {

    int *next = (int *)malloc(len * sizeof(int));
    assert(next); // 确保分配成功

    // 初始化next数组
    next[0] = -1; // next[0] 设置为 -1，表示没有有效前缀匹配
    if (len <= 1) { // 如果模式串长度小于等于 1，直接返回
        return next;
    }
    next[1] = 0; // next[1] 设置为 0，表示第一个字符没有公共前后缀

    // 构建next数组， i 从模式串的第三个字符开始， j 指向当前匹配的最长前缀长度
    int i = 2, j = 0;
    while (i < len) {
        if (needle[i - 1] == needle[j]) {
            j++;
            next[i] = j;
            i++;
        } else if (j > 0) {
            // 如果不匹配且 j > 0， 回退到次长匹配前缀的长度
            j = next[j];
        } else { // j == 0
            next[i] = 0;
            i++;
        }
    }
    return next;
}

int strStr(char* haystack, char* needle) {

    int needle_len = strlen(needle);
    int haystack_len = strlen(haystack);

    int *next = build_next(needle, needle_len);

    int i = 0, j = 0; // i 指向主串的当前起始位置, j 指向模式串的当前匹配位置
    while (i <= haystack_len - needle_len) {
        if (haystack[i + j] == needle[j]) {
            j++;
            if (j == needle_len) {
                free(next);
                next = NULL;
                return i;
            }
        } else { // haystack[i + j] != needle[j]
            i = i + j - next[j]; // 调整主串的起始位置,j的回退也可以看作是子串的右移，
                                 // 由于最终返回的是i，每次匹配失败时主串中的i也要与子串的头对齐，即i+子串右移数(即j回退数)
            j = j > 0 ? next[j] : 0;
        }
    }

    free(next);
    next = NULL;
    return -1;
```

```c
        if(inorder[index] == node->val) break;
    }

    // 递归调用函数构造根节点的左子树（去除根节点）
    node->left = buildTree(preorder + 1, index, inorder, index);

    // 递归调用函数构造根节点的右子树（去除根节点所在的位置，且两数组元素个数始终相等）
    node->right = buildTree(preorder + index + 1, inorderSize - index - 1, inorder + index + 1, inorderSize - index - 1);
    
    return node;
}

/*
索引index：
    当前树的根结点即当前前序数组中的首元素，在当前中序数组中的位置下标索引，并且由中序特点和下标特性可知，正好左边都是左子树结点也就是左子树有index个，那么右子树就是当前中序数组个数-index个-1个(这1个就是index索引上的根结点)
(之后省去“当前”，默认为当前层递归)

node->left四个参数：
1.左子树前序数组首地址：即当前树前序首地址+1(即向右偏移1)
2.左子树前序数组元素个数：即左子树结点数(即index个)
3.左子树中序数组首地址：左子树中序数组首地址一直与中序数组首地址是同一个(即inorder)
4.左子树中序数组元素个数：即左子树结点数(同2.index个)
```

```c
// first 始终保存遍历过的最小值，second 保存比 first 大的最小值，当找到一个比 second 还大的数时，就找到了递增三元组
// 贪心特点：
// 局部最优选择：每一步都维护当前最小的 first 和尽可能小的 second
// 无回溯：一旦做出选择（更新 first 或 second），就不会回头修改
// 为后续创造最优条件：保持 first 和 second 尽可能小，更容易找到第三个数
bool increasingTriplet(int* nums, int numsSize) {
    if (numsSize < 3) return false;
    
    int first = INT_MAX;   // 最小的数
    int second = INT_MAX;  // 第二小的数
    // 注：不可同时将first与second初始化成nums[0]与nums[1]，但first一定是从nums[0]开始变化的
    
    for (int i = 0; i < numsSize; i++) {

            first = nums[i];  // 更新最小值
        } else if (nums[i] <= second) {
            second = nums[i]; // 更新第二小值
        } else {
            return true;      // 找到了比 first 和 second 都大的数
            // 此时存在 first < second < nums[i]
        }
    }
    return false;
}
```

## 21.[[Medium]300. 最长递增子序列](https://leetcode.cn/problems/longest-increasing-subsequence/description/)——”动态规划“

**动态规划（Dynamic Programming, DP）** 是一种通过将原问题分解为相互重叠的子问题，并求解这些子问题来得到原问题解的方法。

**核心思想：**

1. **将问题分解**：把原问题分解成若干子问题
2. **记住过往**：存储子问题的解，避免重复计算
3. **递推求解**：利用子问题的解构建原问题的解

**动态规划的三大特征：**

1. **最优子结构：**一个问题的最优解包含其子问题的最优解。
2. **重叠子问题：**不同的子问题会重复出现，而不是每次都产生新问题。
3. **状态转移方程：**描述子问题之间关系的数学公式。

```c
// dp[i]定义：dp[i]为考虑前[0...i]个元素，以第[i]个数字结尾的最长上升子序列的长度，注意nums[i]必须被选取，注意：以nums[i]结尾，且还是上升子序列，如1,2,3,0就不行——不是上升子序列，以0为结尾的上升子序列只有自身即dp[3] = 1；

int lengthOfLIS(int* nums, int numsSize) {
    if (numsSize == 0)
        return 0;

    // 动态分配dp数组
    int* dp = (int*)malloc(numsSize * sizeof(int));
    if (dp == NULL)
        return -1; // 内存分配失败

    for (int i = 0; i < numsSize; ++i) { // 遍历dp[i]，改变其值
        dp[i] = 1; // 将每个dp[i]都初始化为1即表示最坏结果：本身就是一个最长递增子序列，即前面没有比他小的
        for (int j = 0; j < i; ++j) // 找dp[0...i-1]中最大的值（通过一步步迭代来实现）
            if (nums[j] < nums[i])
                if (dp[j] + 1 > dp[i])
                    dp[i] = dp[j] + 1;
    }

    // 找出最大值
    int maxLen = dp[0];
    for (int i = 1; i < numsSize; ++i)
        if (dp[i] > maxLen)
            maxLen = dp[i];

    free(dp);
    return maxLen;
}
```

## 22.[[Medium+]导弹拦截](https://www.luogu.com.cn/problem/P1020)——时间复杂度更低的”最长递增子序列“——“动态规划”

**问题重述**

题目给了我们一串导弹的高度，需要输出两个数：

1. **一套系统最多能拦截多少导弹**：即**最长不上升子序列**的长度（后面的高度 ≤ 前面的高度）。
2. **拦截所有导弹最少需要多少套系统**：即**最少的不上升子序列覆盖数**。根据 **Dilworth 定理**，这个问题等价于求**最长上升子序列**的长度。

**核心思路与解答**

**第一问：最长不上升子序列长度**

这正是你之前代码中第一部分在做的事。我们维护一个数组 `d`，`d[i]` 表示**长度为 i 的不上升子序列的最后一个元素的最大值**。遍历每个导弹高度 `h`，在 `d` 中二分查找**第一个**小于 `h` 的位置，然后用 `h` 更新它。

**第二问：最少系统数（最长上升子序列长度）**

同样可以用二分贪心。维护一个数组 `g`，`g[i]` 表示**长度为 i 的上升子序列的最后一个元素的最小值**。遍历每个导弹高度 `h`，在 `g` 中二分查找**第一个**大于等于 `h` 的位置，然后用 `h` 更新它。最终 `g` 的长度就是最长上升子序列的长度。

```c++
// 第一问：最长不上升子序列
// 第二问：最长上升子序列的序列个数

#include <iostream>
#include <algorithm>
using namespace std;

const int MAXN = 100005;
int h[100005], n = 0;
int d[MAXN], g[MAXN]; // d用于第一问，g用于第二问

int main() {
    // 读取所有导弹高度
    while (cin >> h[++n]);
    n--; // 因为最后读入了EOF，总数要减1

    // --- 第一问：最长不上升子序列 ---
    int len1 = 0;
    d[0] = 50001; // 初始化为一个大于最大高度的值
    for (int i = 1; i <= n; i++) {
        int l = 0, r = len1 + 1;
        while (l + 1 < r) {
            int mid = (l + r) / 2;
            if (d[mid] >= h[i]) l = mid; // 找最后一个 >= h[i] 的位置
            else r = mid;
        }
        int x = l + 1;
        if (x > len1) len1 = x;
        d[x] = h[i];
    }
    
    // --- 第二问：最长上升子序列（求最少系统数）---
    int len2 = 0;
    g[0] = 0; // 初始化为0
    for (int i = 1; i <= n; i++) {
        int l = 0, r = len2 + 1;
        while (l + 1 < r) {
            int mid = (l + r) / 2;
            if (g[mid] < h[i]) l = mid; // 找最后一个 < h[i] 的位置
            else r = mid;
        }
        int x = l + 1;
        if (x > len2) len2 = x;
        g[x] = h[i];
    }

    cout << len1 << endl;
    cout << len2 << endl;

    return 0;
}
```

## 23.[[Medium+]368. 最大整除子集](https://leetcode.cn/problems/largest-divisible-subset/description/)

```c
int cmp(const void *a, const void *b){
    return *(int*)a - *(int*)b;
}

/*
补充：整除关系的传递性：如果a|b，并且b|c，那么a|c，可知：
1.如果整数a是整除子集S1的最小整数b的因数（即a|b），那么可以将a添加到S1中得到一个更大的整除子集；
2.如果整数c是整除子集S2的最大整数d的倍数（即d|c），那么可以将c添加到S2中得到一个更大的整除子集

这个传递性也正是这题可以进行动态规划的原因，也使状态转移变得可实现且简单实现
*/

int* largestDivisibleSubset(int* nums, int numsSize, int* returnSize) {
// 先排序，排序之后判断一对整除数，只需要判断后者也就是大的能不能被小的整除，因为小除大一定不整除
    qsort(nums, numsSize, sizeof(int), cmp);
/*——————此题中最大整数子集的个数易求，但子集内容难寻
动态规划标准步骤：
1.状态定义：dp[i]表示在nums升序时，以nums[i]为最大整数即为末尾元素的最大整数子集的大小(nums[i]必选)
2.状态转移方程：遍历数组，一层循环范围i=0...numsSize-1，二层循环范围j=0...i-1，如果所以nums[j]均能整除nums[i]（意为nums[i]%nums[j]=0），说明nums [i]可以加入以nums[j]为最大整数的整除子集，以此来构成一个更大的整除子集。在这个过程在每次nums[i]与nums[j]比较时都会尝试更新当前i对应的dp[i]的值，当一个一层循环结束时即i更新时就确定了一个不变的dp[i]。
3.初始化：由于nums[i]必须被选择，因此对于任意i=0...numsSize-1，初始化的时候dp[i]，意为最坏的情况就是仅nums[i]构成以他为末尾的升序的整除子集
*/  
    // int dp[numsSize]; //C中不支持变长数组
    int *dp = (int*)malloc(numsSize*sizeof(int));
    for(int i = 0; i < numsSize; i++)
        dp[i] = 1;

    // 动态规划找出最大子集的个数、最大子集中的最大整数
    int max_size = 1; // 目前情况下的最大子集的元素个数————倒推时为待加入最大子集的元素个数
    int max_val = nums[0]; // 当前找到的最大子集中的最大值————倒推时为已加入最大子集中的最小值
    for(int i = 0; i < numsSize; i++){
        for(int j = 0; j < i; j++){
            if(nums[i] % nums[j] == 0)
                dp[i] = fmax(dp[i], dp[j] + 1); // 题目中说没有重复元素很重要,避免了混乱情况
        }
        if(dp[i] > max_size){
            max_size = dp[i];
            max_val = nums[i];
        }
    }

    // 倒推得最大子集
    int* res = malloc(sizeof(int) * numsSize); // 避免装不下，直接申请与原数组相同大小的空间
    *returnSize = 0;
    // 若是只存在长度为1的最大子集，则任选一个元素即可
    if(max_size == 1){
        res[(*returnSize)++] = nums[0]; 
        return res;
    }
    // 最大子集长度大于1即正常情况
    for(int i = numsSize - 1; i >= 0 && max_size > 0; i--)
        if(dp[i] == max_size && max_val % nums[i] == 0){ // 判断哪个值是真该加入 全局的最大子集
            res[(*returnSize)++] = nums[i];
            max_val = nums[i]; // 更新当前已加入最大子集中的最'小'值
            max_size--; // 更新待加入最大子集的元素个数
        }
    
    return res;
}
```

**==!!!只要涉及到改变head的问题(包含但不仅限于删除)，就要有哨兵结点，不然多考虑一个特殊处理逻辑，非常麻烦==**

**法一：常规迭代，逻辑混乱，迭代指针pre处于操作范围之内即pre并非是pre而变成cur，cur变为了next——一般**

```c
struct ListNode* swapPairs(struct ListNode* head) {
    if(head == NULL || head->next == NULL) // 无结点或者只有1个结点直接返回head
        return head;
    struct ListNode* pre = head;
    struct ListNode* cur = head->next;
    head = cur; // 更新head变为原链表中第二个结点

    while(pre && cur){ // 每一对结点的第一个与第二个
        struct ListNode* t = cur->next; // 此辅助指针设置完美
        cur->next = pre;
        // pre->next的更新要分两种情况
        if( t && t->next)
            pre->next = t->next;
        else
            pre->next = t; // 正好只有两个结点时，pre->next指向NULL
        pre = t;
        if(t)
            cur = t->next;
    }

    return head;
}
```

**法二：带哨兵结点的迭代——较优**

```c
struct ListNode* swapPairs(struct ListNode* head) {
    struct ListNode dummy = {0, head}; // 用哨兵节点简化代码逻辑
    struct ListNode* node0 = &dummy;
    struct ListNode* node1 = head;
    while (node1 && node1->next) { // 至少有两个节点
        struct ListNode* node2 = node1->next;
        struct ListNode* node3 = node2->next;

        node0->next = node2; // 0 -> 2
        node2->next = node1; // 2 -> 1
        node1->next = node3; // 1 -> 3 —— 由于哨兵的存在，node1不用考虑指向node4了，因为每次循环的初始状态都类似，												不会有特殊情况，这就是哨兵的作用，统一迭代操作逻辑
        node0 = node1; // 下一轮交换，0 是 1
        node1 = node3; // 下一轮交换，1 是 3
    }
    return dummy.next; // 返回新链表的头节点
}
```

**法三：递归——最优**



```c
/*
这里直接用 swapPairs 当作递归函数：
1.递归边界：
如果 head 或者 head->next 为空，说明剩余节点不足两个，无需交换，返回 head。
2.递归操作：
先交换以node3为头节点的链表，即递归调用 swapPairs(node3)。
把 node1指向递归返回的链表头。
把 node2指向 node1。
3.返回值：
返回node2，作为交换后的链表头节点。
*/
// 这题则是先操作，再递归
struct ListNode* swapPairs(struct ListNode* head) {
    if (head == NULL || head->next == NULL) {
        return head;
    }

    struct ListNode* node1 = head;
    struct ListNode* node2 = head->next;
    struct ListNode* node3 = node2->next;

    node1->next = swapPairs(node3); // 1 指向递归返回的链表头
    node2->next = node1; // 2 指向 1

    return node2; // 返回交换后的链表头节点
}
```

```c
int maxArea(int* height, int heightSize) {
    int l = 0, r = heightSize - 1;

    int ans = (r - l) * min(height[l], height[r]);
    while(l < r){
        if(height[l] == height[r]){
            l++;
            r--;
        }
        else if(height[l] < height[r])

        else
            r--;
```

# ==二叉树的前序、中序、后序遍历的三种实现方式==

1. **递归**

   ​		没什么好说的，根据**根、左、右的相对位置**，分别在递归函数中安排其对应语句的顺序（根为遍历语句即进入数组，左、右对应递归语句）

2. **迭代**

   ​		**前序、中序、后序遍历**的迭代算法使用的辅助空间均为**栈**，而**层序遍历**使用**队列**

   ​		最大的**特点**是：**只存在一句遍历语句即进入数组的语句**
   ​		关于语句的：根依旧对应遍历语句，左、右对应迭代语句

3. **Morris算法**——难度：前序 ＜ 中序 ＜＜ 后序

   ​		==**Morris 遍历的核心思想是利用树的大量空闲指针，实现空间开销的极限缩减**==
   ​		==**Morris 其实解决了一个常规循环中循环到叶子节点后难以回到根节点的问题**，即Morris的核心就是重新回到根结点==

   ​		神中神，时间为O(2n)=O(n)，空间为O(1)。
   ​		**存在两条遍历语句**且之间有关联，分为主遍历语句与次遍历语句，次遍历语句是主遍历语句的预处理

4. 关于三种遍历序列的特点

   ```
   先序: 访问根 → 递归左 → 递归右
          ↑
       第一个访问的就是根
   中序: 递归左 → 访问根 → 递归右
                    ↑
       第一个访问的是最左下的节点（左递归最深处）
   后序: 递归左 → 递归右 → 访问根
                              ↑
       第一个访问的是左递归最深且右子树为空的节点
   ```

   - **先序遍历的第一个结点一定是根结点**
   - **中序遍历的第一个结点一定是最左下角的结点（不一定是叶节点）**
   - **后序遍历的第一个节点一定是叶节点**

5. 补充：**层序遍历**的**辅助队列**大小应该如何设置

   只有在**完全二叉树中**，最后一行的节点数才可能最多。对于有 **`n`** 个节点的完全二叉树：

   - 如果树是满二叉树，最后一行的节点数 = 总节点数的一半（向上取整）

   - 具体公式：当树接近满二叉树时，最后一行节点数 ≈ **`n/2`**（向上取整）

     ```c
     证明:以n=2000举例
     	n = n0 + n1 + n2
     	n = n1 + 2n2 + 1
     联立得：n0 = n2 + 1
     假设n1最少，则n0+n2为2000或者近似于2000，所以最多只有n0 = 1000，n2有999个，还有一个n1结点
     ```

```c
3.从给定节点开始，沿着右孩子向下遍历，将路径上的节点值顺序存入数组，最后将这一段逆序（实现后序遍历的效果）

    while(node){
        ans[(*returnSize)++] = node->val;
        node = node->right;
        count++;
    }
    for(int l = *returnSize - count, r = *returnSize - 1; l < r; l++, r--){
        int t = ans[l];
        ans[l] = ans[r];
        ans[r] = t;
    }
```

```c
typedef struct{

    int front;              // 队首指针，指向队首元素

    q->size = 0;

    // 如果树是满二叉树，最后一行的节点数 = 总节点数的一半（向上取整）
    // 具体公式：当树接近满二叉树时，最后一行节点数 ≈ n/2（向上取整）

    q->size--;
    return node;
}

    }

            if(curLevelcurNode->right)

    *returnSize = index; // 最终的层索引就是总行数，因为每处理一层index就++，所以一直代表层数

    free(q);       // 辅助队列本体

    return ans;
}
```

# ↓==二分查找==

​		二分查找一般是插入排序的前置，所以最终的位置就是待插入元素最终在数组中的位置即索引

​		所以这里涉及到一个上下边界的问题，因为若是待插入数组中已经存在与待插入元素相同值的元素值，新元素则可以出现在这些元素值的任一个位置和它们的左/右偏移一个的位置，而lower_bound则是寻找下边界——最终位置左边的所有元素均小于其本身-右边则大于或者等于，upper_bound则是寻找上边界——最终位置右边的所有元素均大于其本身-左边小于或者等于

​		这里讨论的则是lower_bound，即寻找第一个(最左边的)大于或者等于待插入元素的值，这个值的下标就是我们要查找的新元素的下标，一共有包含**闭区间**、**左闭右开区间**和**开区间**三种写法。

```c

	答：这种写法适用范围很窄。在有多个等于 target 的数的情况下，如果在二分的过程中立刻返回，我们得到的可能不是第一个等于 target 的数的下标。为什么返回第一个等于 target 的数的下标更好呢？因为这可以解决更复杂的题目，比如给你一个有序数组 [1,1,3,3,3,3,5]，让你计算有多少个数小于 3。我们可以二分查找第一个等于 3 的数的下标 2，那么下标小于 2 的数就是元素值小于 3 的数，这有 2 个。对比可以发现，如果在二分中途就立刻返回，我们不一定找到的是第一个等于 3 的数的下标，所以不一定能算出正确答案 2。
```

```c
    while (left + 1 < right) { // 区间不为空
        // 循环不变量：
        // nums[left] < target
        // nums[right] >= target
        int mid = left + (right - left) / 2; // 注意括号内不能为负，所以大减小即右减左
```

```
	问：如何理解 end = lowerBound(nums, target + 1) - 1 这段代码？
	答：要想找到 ≤target 的最后一个数，无需单独再写一个二分。我们可以先找到这个数的右边相邻数字，也就是 >target 的第一个数。在所有数都是整数的前提下，>target 等价于 ≥target+1，这样就可以复用我们已经写好的二分函数了，即 lowerBound(nums, target + 1)，算出这个数的下标后，将其减一，就得到 ≤target 的最后一个数的下标。

	问：如果 ≥target+1 的第一个数不存在怎么办？
	答：这说明数组中的数都 ≤target。如果数组中有 target，那么数组的最后一个数（下标 n−1）就是 target（因为数组是递增的）。同时，lowerBound(nums, target + 1) 在这种情况下会返回 n，减一得到 n−1，这正是我们要计算的下标。
```

  ```
  贪心算法是动态规划的一个特例：
  	动态规划：考虑所有可能的决策路径，通过状态转移找到全局最优
  	贪心算法：在每一步只考虑一个"看起来最优"的决策，不回溯
  	
  如果贪心算法能正确解决问题，说明该问题满足：
  	最优子结构：全局最优包含局部最优
  	贪心选择性：局部最优选择一定能导向全局最优
  	
  这两个性质同时满足时，DP 当然也能解决（因为 DP 只需要最优子结构）。
  ```

# ==什么是记忆化搜索(Memoization)==

记忆化搜索是一种**优化递归算法**的技术，核心思想是：**用空间换时间**，将已经计算过的结果存储起来，避免重复计算。

![image-20260325192405715](D:\Typora Picture\image-20260325192405715.png)

可以看到 `recursive(3)`、`recursive(2)` 等被重复计算了多次。

**记忆化搜索的做法**：第一次计算 `recursive(3)` 后，把结果保存起来，下次再需要时直接返回，不再递归。

```c
// 法三：位运算-模3加法——Hard
/*
a与b所代表的含义：
    首先以模2加法为例（136.），仅每位需要1个比特来表示其相应比特出现的次数（只考虑有效值即1，0不需要考虑，因为0无论出现几次其和也为0，并且也不能表示有效数值），例如现在有个数比如3，它的最低有效位是1，它出现1次，那么其模ans(初始为0，即所以位均为0)的最低位0就是1即代表着它出现了1次，所以在返回时这个1也能代表是它的本身数值，但若是它出现第2次，再度模（此时模的是1）的结果是0，即0就代表它出现了2次，因为这是模2加法，所以出现2次就是不能对结果产生影响，即结果上的各个位都不能代表它的本身数值，也就是说与它无关了（上述那么多内容都是拿1个位即最低有效位举例的，在实际运算中，模运算^是能同时进行所有位的相应模运算的，均同理）
    通过模2加法的讲述，我们可以知道1个比特位表示两种状态，分别代表出现0次与1次。所以不难推出，若是现在想要表示3个状态即分别是0次、1次、2次，则需要至少2个比特位，a、b的值为00、01、10时分别对应0次、1次、2次，11搁置不用。并且模3意为出现3次代表没出现即体现不到最终的结果上，那么现在因为要两个比特位，即需要2个数来表示结果，所以最终选哪个数呢，因为出现0次与1次是无效的，所以正好2个数代表出现1次与2次的有效值，且由模2中0无效1有效，不难推出当出现1次时是01即b中1是有效，出现2次时是10即a中1是有效的，所以找唯一一个出现1次的数九返回b，出现2次返回a（前提是其余均出现0次(废话)或者3次）

                            0	            0	    0
                            1	            0	    1
                            2	            1	    0

    1.当 a=0 且 b=0 时，a 必须保持不变，仍然为 0。

int singleNumber(int* nums, int numsSize){
    int a = 0, b = 0;
    for(int i = 0; i <numsSize; i++){
        int tmp_a = a;
        a = (a ^ nums[i]) & (a | b);
        b = (b ^ nums[i]) & ~tmp_a;
    }
```

```c
        } else {
            type2 ^= num;
        }
```

## 78.[[Medium]15. 三数之和](https://leetcode.cn/problems/3sum/description/)——“双指针-第三个指针用来切换数组”

```c
/*
「不重复」的本质是什么？我们保持三重循环的大框架不变，只需要保证：
    第二重循环枚举到的元素不小于当前第一重循环枚举到的元素；
    第三重循环枚举到的元素不小于当前第二重循环枚举到的元素。
    也就是说，我们枚举的三元组 (a,b,c) 满足 a≤b≤c，保证了只有 (a,b,c) 这个顺序会被枚举到，而 (b,a,c)、(c,b,a) 等等这些不会，这样就减少了重复。要实现这一点，我们可以将数组中的元素从小到大进行排序，随后使用普通的三重循环就可以满足上面的要求。
*/

// 本质双指针，i用来更换数组，即每次i的更新代表切换到了一个新的数组，i就是这个数组的头结点(索引-1)，然后j与k分别再初始化为每个新数组中的左右两端双指针开始夹逼
int cmp(const void* a, const void* b){
    return *(int*)a - *(int*)b;
}

int** threeSum(int* nums, int numsSize, int* returnSize, int** returnColumnSizes) {
    qsort(nums, numsSize, sizeof(int), cmp);
    int** ans = (int**)malloc(numsSize*numsSize * sizeof(int*)); // 足以容纳答案，当整数 n=3,4,5,6,7,8 时，C-n-3<n2; 当 n≥9 时，C-n-3>n2
    *returnColumnSizes = (int*)malloc(numsSize*numsSize * sizeof(int));
    int m = 0;
    for(int i = 0; i < numsSize - 2; i++){ // 即只遍历到倒数第三个元素
        int x = nums[i];
        if(i > 0 && x == nums[i - 1])   continue; // 跳过重复数字
        if(x + nums[i + 1] + nums[i + 2] > 0)   break; // 继续往后只会比0越来越大，终止遍历，后面不会再有了
        if(x + nums[numsSize - 2] + nums[numsSize - 1] < 0)   continue; // 当前数太小了，与最大两个相加还<0，直接跳过，开始下次遍历
        // 通过上面三次判断后能来到这里，则代表可能存在三元组，开始寻找
        int j = i + 1, k = numsSize - 1; // 以当前i为索引-1的数组开始设置左右双指针开始遍历
        while(j < k){
            int s = x + nums[j] + nums[k];
            if(s > 0)
                k--;
            else if(s < 0)
                j++;
            else{ // 即三数之和为0，添加三元组(x, nums[j], nums[k]) 到 ans 中
                int* tuple = (int*)malloc(3 * sizeof(int)); // 必须得用堆，不能用栈即int tuple[3]，因为需要传出这些数据
                tuple[0] = x;
                tuple[1] = nums[j];
                tuple[2] = nums[k];
                ans[m] = tuple;
                (*returnColumnSizes)[m++] = 3;
                // 跳过重复数字
                for(j++; j < k && nums[j] == nums[j - 1]; j++);
                for(k--; j < k && nums[k] == nums[k + 1]; k--);
            }
        }
    }
    *returnSize = m;
```

```c
// 法二：双指针-相向双指针即左右指针——对法一空间上的优化
/*
从法一可知：对于每一个水桶，我们都是找左右两边的最大值中的那个最小值，也就是说每次都是比较两个数，那么关于这一点有没有什么性质我们还未发现呢，可以注意到在两个最大值数组的迭代中，元素都是非严格递增的，并且前缀后缀最大值中总是较小的那一个起决定性作用，也就是说每次选择当前部分更新的两个最大值中的较小值，对于它所处的位置来说，答案是不是已经确定了呢，可以验证一下：以倒数第三个(j)元素举例，假设左边已经遍历到最高点(即倒数第五个元素)即当前的pre_max(对于j来说未更新完)，也就是说这个桶左高为3，它的右高即当前的suf_max为2（注意：此时的suf_max是已经更新完了的），也就是说它的高度已经注定了即由较小的suf_max=2决定，因为pre_max只会变大不会变小，而suf_max对于j来说已经更新完了，再往左走就与j没有关系了。——从而发现了一条规律：当前的两个更新结果中的较小值对于它所处的位置的那个元素来说是已经更新完了的，并且又是起决定性作用的较小值，所以这个桶的接水量已经注定。
*/
#define MY_MAX(a, b) ((a) > (b) ? (a) : (b))
// #define MY_MIN(a, b) ((a) < (b) ? (a) :(b))
int trap(int* height, int heightSize) {
    int ans = 0, pre_max = 0, suf_max = 0;
    int left = 0, right = heightSize - 1;
    while(left < right){
        pre_max = MY_MAX(pre_max, height[left]);
        suf_max = MY_MAX(suf_max, height[right]);
        ans += pre_max < suf_max ? pre_max - height[left++] : suf_max - height[right--]; // 不用取最小值的宏是为了比较的同时还能精确的选中指向较小值的那个指针使其更新
    }

    return ans;
}
```

```c
// 法三：单调栈
/*
单调栈与双指针法的不同在于计算的方式，双指针法是纵向计算的，而单调栈法是横向计算的
*/
#define MY_MIN(a, b) ((a) < (b) ? (a) :(b))
int trap(int* height, int heightSize) {
    int ans = 0;

    // 用数组模拟栈，栈大小最多为heightSize
    int *st = (int*)malloc(heightSize * sizeof(int));
    int top = -1; // 栈顶指针，一直指向栈顶元素(栈元素为数组元素的下标)，初始为空栈

    for(int i = 0; i < heightSize; i++){
        int h = height[i];
        while(top >= 0 && height[st[top]] <= h){ // 即这是一个单调不增栈，只有当前元素大于或等于栈顶元素对应的元素时，才会开始处理，且在处理结束时当前元素的下标会入栈作为新的单调起点（也即并非全程单调）
            int bottom_h = height[st[top--]]; // 出栈
            if(top < 0)
                break;
            int left = st[top];
            int h = MY_MIN(height[left], height[i]) - bottom_h; // 此次计算的面积的高
            ans += h * (i - left - 1);
        }
        st[++top] = i; // 当前元素的下标入栈，作为新的单调起点
    }
    
    free(st);
    return ans;
}
```
