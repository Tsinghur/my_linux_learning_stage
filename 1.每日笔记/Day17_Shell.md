3. 递归的搜索家目录下的所有文件，搜索文件内容中所有以**f或F开头**，然后以**t结尾**的**单词**，展示出行号。

   注意：所谓单词，要以空格或制表符来进行分隔。

   ```c
   grep -rn "\<[Ff][^ \t]*t\>" ~
   \<、\>：分别匹配一个单词的开始和结束。单词同样是基于空格、制表符间隔去判断的。
   *：重复任意次数，可以是零次，一次或多次。
   ^在[]中代表非的意思
   []代表或,例如[abc]代表可匹配a或b或c
   ```

4. 管道符|与正则表达式与grep搜索文件内容：

   ```c
   // 这条命令是 Linux/Unix 下的经典组合用法，核心作用是：在当前目录及其所有子目录的 .c 源代码文件中，精准查找包含 main( 函数的行，并显示行号。
   find . -name "*.c" | xargs grep -nE "\<main\("
   
   /*
   "\<main\("
   搜索的正则表达式，精准匹配「main (」：
   	\<：单词开头锚点（避免匹配 amain( 等）
   	main\(：匹配 main(（( 是正则元字符，加 \ 转义）
   	整体：只匹配以 main 开头、紧跟 ( 的内容，即 main() 函数的定义
   */
   ```

5. Linux硬链接：

   - **不可以给目录文件创建硬链接**

     如果允许手动给目录创建硬链接，会出现「A 目录硬链接到 B 目录，B 目录又包含 A 目录」的循环引用，导致 `ls`、`find` 等命令遍历文件系统时陷入死循环，甚至破坏文件系统结构。

   - **但是目录文件的硬链接数是会改变的**

     Linux 文件系统中，每个目录默认有两个硬链接：`./`（指向自己）和 `../`（指向上级目录），例如：假设现在存在一个目录：`/dir1`，它下面有两个子目录 `/dir1/dir2 `和 `/dir1/dir3`，在 `/dir1` 下创建一个新的子目录 `/dir1/dir4`，此时/dir1目录文件的硬链接数+1，因为新的子目录当中".."指向上级目录dir1

6. **==Linux通配符==**

   1. `*`：表示匹配**0个**或者**多个**任意字符
   2. `?`：表示匹配**1个**任意字符
   3. `[...]`：根据中括号内的字符进行匹配。
      1. [characters] 表示匹配中**括号内的**任意**一个**字符
      2. [!characters] 表示匹配中**括号外的**任意**一个**字符

   ```c
   假如当前工作目录下存在以下子目录：
   dir dir1 dir2 dir3 dira dirb dirc diraaa dirbbb dirccc a b c dirx diry dirz
   以下说法正确的是？
   A.rmdir dir[abc] -该指令会删除dira dirb dirc
   B.rmdir dir[!abc] -该指令会删除dir dir1 dir2 dir3 dirx diry dirz
   C.rmdir dir* -该指令不会删除dir目录
   D.rmdir * -该指令会删除所有的目录
   正确答案: AD
   // B中不会删除dir，因为'[]'必定会匹配"一个"字符
   // C中会删除，因为'*'可以匹配"0个"字符
   ```

7. Linux**目录**的权限

   1. **执行权限**与**ls指令**

      | 目录权限组合                         | `ls/ll` 能否执行       | 具体表现                                        |
      | ------------------------------------ | ---------------------- | ----------------------------------------------- |
      | 仅有读权限（r）、==无==执行权限（x） | ❌ 无法正常执行         | ==报错「权限不足」，连目录内容都列不出来==      |
      | ==仅有==执行权限（x）、无读权限（r） | ✅ 能执行，但看不到内容 | ==不报错，但无法列出目录里的文件 / 文件夹名称== |
      | 同时有读（r）+ 执行（x）权限         | ✅ 完全正常执行         | 能列出目录所有内容（符合 `ls/ll` 预期）         |

   2. **写权限**

      1. ==影响用户能否在目录中创建、删除、重命名文件==
      2. 影响cp、mv、mkdir、rm等指令的执行

   3. **目录的读写权限都依赖执行权限，没有执行权限，目录即便有读写权限也没有意义**

   4. (不完全准确)即便目录文件没有写权限，但只要文件具有写权限，那么就可以修改文件内容
      目录的 x 权限是「进门的钥匙」，文件的 w 权限是「修改内容的权限」—— 有钥匙 + 有权限，才能改内容；没钥匙，再有权限也进不去门。

8. Linux**普通文件**的权限

   ```c
   在Linux系统中，关于普通文件的权限相关的说法:
    
   ❌A.拥有写权限（w）允许用户删除文件。 // 删除文件是看文件所在目录有没有w权限
   ❌B.执行权限（x）不影响脚本文件，只适用于二进制程序文件。 // 对两者都重要
   ❌C.拥有写权限（w）允许用户修改文件的权限和所有权。 // 只有文件所有者和root能做，能改内容，不代表能改权限/所有者
   D.读权限（r）关系到用户是否能够阅读文件内容。
   E.写权限（w）关系到用户是否能够修改文件内容。
   F.执行权限（x）关系到用户能否将文件运行。
   G.重命名一个文件，这个操作和文件本身的权限没有关系。和它父目录的权限有关系
   ```

9. **在Linux系统中，关于重定向的说法：**

   ```c
   A.重定向允许改变进程的标准输入流的输入源以及标准输出和标准错误流的输出目的地
   ❌B.通过重定向可以使得原本从标准输入读数据的进程，直接从文件中读数据，而不是从stdin缓冲区中读 // 重定向本质还是从stdin																					读，只不过从直接变成了间接
   ❌C.<是重定向标准输出的重定向符号 // 重定向标准输入
   D.echo "test" >> file会将数据追加写入到文件file中 // >>是追加重定向，>是覆盖重定向
   E.一般很少重定向标准输入，更多的是重定向标准输出和标准错误
   G.cat file 这个指令并没有从标准输入stdin中读数据，而是直接从文件中读数据 // 这个与重定向无关，直接从文件读数据
   ```

11. ```
    find . -type d -empty -delete
    ```

    - 解释
      - `find . -type d`：查找当前目录下所有子目录
      - `-empty`：只匹配空目录
      - `-delete`：删除匹配到的空目录
