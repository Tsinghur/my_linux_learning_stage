## Day22——文件映射

### 相关函数

| void *mmap(void *addr, size_t length, int prot, int flags, int fd, off_t offset); | #include <sys/mman.h>                      | 函数会返回一个指向用户态空间映射文件内容的指针，利用该指针操作映射的内存区域，就可以实现操作文件内容 |
| :----------------------------------------------------------- | :----------------------------------------- | :----------------------------------------------------------- |
| int munmap(void *addr, size_t length);                       | #include <sys/mman.h>                      | 用于解除一个映射区域，释放内存资源                           |
| int fstat(int fd, struct stat *buf);                         | \#include <sys/stat.h> #include <unistd.h> | 通过已打开的文件描述符fd获取文件的详细属性信息               |
| void *memcpy(void *dest, const void  *src, size_t n);        | \#include <string.h>                       | 从指定内存地址拷贝指定字节数的数据到另一块内存地址           |
| int fflush(FILE *stream);                                    | \#include <stdio.h>                        | 强制将缓冲区中未写入的数据刷新到对应的输出流 / 文件中        |
| int dup(int oldfd);                                          | #include <unistd.h>                        | 复制文件描述符使其与原始文件描述符指向同一个文件对象         |

注：

- mmap失败时返回：(void*)-1
- 即使不接收返回值，dup也会自动申请一个新的fd且其与oldfd指向同一个文件对象，但是二者值肯定不同，且fd为当前值最小的可用文件描述符
- close与dup都是以文件描述符fd与文件对象之间的**连接**为操作对象的，不会直接操作文件对象或者文件描述符，例如close(1)只是关闭值为1的fd与其对应文件对象之间的**连接**，而不是关闭文件对象或者与文件对象所连接的所有文件描述符，并且也只关闭**一条**连接

1. ==**mmap 根本不使用文件描述符的“当前读写位置**，而`read/write` **强依赖文件偏移量**”==
   也就是说在mmap复制文件时，通过lseek获取文件大小后，根本不需要指针的倒带操作，**但是read/write需要**

2. **==缓冲区==**

   1. **缓冲区的三种类型**

      | 类型   | 触发输出条件        | 典型对象 |
      | ------ | ------------------- | -------- |
      | 全缓冲 | **缓冲满 / fflush** | 文件     |
      | 行缓冲 | **遇到 `\n`**       | 终端     |
      | 无缓冲 | 立即输出            | stderr   |

   2. **stdout 的“动态变化”**

      第一次使用stdout时的上下文（它==**指向的**==是终端还是文件）将决定stdout的缓冲区刷新机制，并且在整个进程生命周期，这个刷新机制将默认保持不变：==（这个指向是标准输出流的文件描述符STDOUT_FILENO即fd=1的指向）==

      - 如果第一次使用stdout时它**指向一个终端**（比如用printf），此时缓冲区就是行缓冲的。
      - 如果第一次使用时stdout**被重定向到文件或其他非终端设备**，缓冲区就变成了满缓冲区，这意味着缓冲区会积累输出直到满了或者手动刷新。

   3. **fflush**——把“用户态缓冲区”的数据，强制写入到内核（文件描述符）

      ```c
      // printf → stdout缓冲区（用户态） → write(fd) → 内核缓冲（page cache） → 磁盘/终端
      ```

   4. **行缓冲**如果一直不遇到 `\n`，确实可能一直不刷新，**但是在以下情况仍会刷新：**

      1. **缓冲区满**
      2. 手动 fflush
      3. **程序结束**
      4. **发生输入操作时**（在进行输入操作前，C 标准库会自动刷新行缓冲的输出流，这是为了保证提示信息能够在程序阻塞等待输入之前显示出来，从而保证良好的交互体验，“程序要等你输入之前，必须先把话说完”）

   5. **补充：程序“正常结束”时会自动刷新(任何种类)缓冲区，但“异常结束”时不一定**
