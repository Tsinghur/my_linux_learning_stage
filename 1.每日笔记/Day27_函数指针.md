## Day27——函数指针

1. C 语言里的**函数名**，会**自动、智能**地根据上下文，被编译器解析成两种东西：

   1. **函数本身**（调用执行）
   2. **函数指针**（地址，不执行）

   这是 C 语言的**隐式转换规则**，不用你手动写，编译器自动搞定。

2. **函数指针的声明问题**

   1. **【形参】必须声明**

      谁接收函数地址，谁就要**提前声明**：“我要接收一个函数指针！”

      ```c
      //  signal 函数内部早就写好了：第二个参数是 函数指针
      void signal(int sig, void (*func)(int)); 
      ```

   2. **【实参】不用声明**

      谁把地址传过去，谁**直接写函数名**就行，C 语言自动转指针：

      ```c
      // 直接传 my_handler，不用写成指针
      signal(SIGINT, my_handler); 
      ```

3. 函数指针**声明时，绝对不能同时写函数体**

   1. 函数指针 = **指针变量**

      它只是一个**存地址的变量**，用来指向别的函数

      ```c
      // 函数指针声明（只有类型，没有函数体）
      int (*pFunc)(int, int);
      ```

   2. 函数 = **有代码、有函数体**

      它是一段真正可执行的代码

      ```c
      // 函数定义（有函数体）
      int add(int a, int b) {
          return a + b;
      }
      ```
